Why it's worth installing

Four things that actually matter.

Speed

The page stops fighting Safari

The win is work that never happens. Off-screen comments, thumbnails and sidebar rows skip layout entirely via content-visibility. Animations, transitions and shadows are killed outright, so there is nothing to composite each frame. Hover-preview video players — a real <video> element, with media buffers, per thumbnail — are deleted the moment they spawn.

Design

It looks like it belongs on a Mac

The system typeface, macOS focus rings, sidebar selection pills and translucent glass surfaces — with real blur reserved for small surfaces where Safari renders it cheaply. Dark and light both. The Google look is gone.

Parity

You lose nothing

This is the hard part, and it's done. Quality, speed, caption languages, chapters, seek-preview thumbnails, playlists, autoplay, PiP, live streams and DVR scrubbing, stats-for-nerds, every keyboard shortcut — all of it still works.

Memory

The tab stops growing

Comments, replies and the related sidebar are capped, so an hour-long session doesn't leave you with a tab holding thousands of nodes and a dozen buffered video elements. Shorts redirect to the normal watch page, where you get a real scrubber.

Nothing removed

Everything you'd miss, still there.

Install it.

iTube is a userscript — a single file, no build step, no tracking, no network calls of its own. It needs a userscript manager to run.

Safari · recommended

Userscripts

  1. Install Userscripts from the Mac App Store — it's free.
  2. Open Safari → Settings → Extensions and enable it, then allow it on youtube.com.
  3. Click the puzzle icon in the toolbar, choose Open app, and set a scripts folder.
  4. Paste the script from the link below into a new file ending in .user.js.
Chrome, Edge, Firefox

Tampermonkey

  1. Install Tampermonkey (or Violentmonkey) from your browser's extension store.
  2. Open the raw script link below — the manager offers to install it automatically.
  3. Confirm, then reload youtube.com.
  4. Updates arrive on their own: the script carries an update URL.
