~20 versions of features

The biggest things it does.

Everything below ships in the same single file, with no settings screen buried three menus deep.

Real cinema theater mode

The video goes near-fullscreen, everything else fades away, and an ambient glow spills off the frame in real time — not a static border tint.

Zero ads, anywhere

No video ads, no feed ads, no masthead. iTube never renders YouTube's UI, so there's nothing for an ad slot to attach to.

SponsorBlock, built in

Sponsor segments auto-skip with no extension to install and no separate permission prompt — it's just how the timeline works.

Persistent floating mini-player

Keep watching while you browse the rest of the app — it survives back/forward navigation instead of resetting playback.

Command palette — ⌘K

Jump anywhere, run any action, without leaving the keyboard. One shortcut, every destination.

Searchable transcript panel

Full transcript alongside the video — search it, click any line, and the player seeks straight there.

A–B loop & speed up to 5×

Loop any section for practice or transcription, and push playback speed well past YouTube's native ceiling.

Volume boost past 100%

A WebAudio gain stage for the videos that are recorded quiet, with loudness-normalized sliders that don't drift.

Audio-only background mode

Drop the video and keep the sound running — for podcasts, lectures, and background music without the decode cost.

Accent themes & a real settings page

Pick your own accent color site-wide from a proper Settings view in the sidebar — no more digging through a popup.

Mute channels & keywords

Filter out channels or keywords across every feed, and hide videos you've already watched — for real, everywhere.

Optimistic instant likes

Like, dislike and Watch Later update the instant you click — no waiting on a round-trip to feel responsive.

Not a reskin

Why it's actually faster.

The speed isn't a trick applied on top of YouTube's page. It's a different architecture underneath.

1

Its own lean DOM, not YouTube's 20k+ nodes

iTube parks YouTube's real UI offscreen and never touches it. Nothing gets scrolled, hovered or brought into view, so YouTube's comments, related rail and feed rows never lazy-render at all — the work simply never happens.

2

Fully client-side navigation

Every route — home, watch, search, channel, playlist — swaps in-place. No full page reload, no flash of YouTube's own shell on the way in.

3

Windowed feeds

Long feeds render only what's near the viewport instead of materializing every card up front, so scrolling stays cheap no matter how far down you go.

4

Zero frameworks, zero dependencies, one file

No React, no bundler, no npm tree shipped to your browser. Vanilla JS in a single .user.js file that a userscript manager runs directly.

Install it.

iTube is a userscript — a single file, no build step, no dependencies, no tracking. It talks to YouTube's own API and nothing else. It needs a userscript manager to run.

Safari · recommended

Userscripts

  1. Install Userscripts from the Mac App Store — it's free.
  2. Open Safari → Settings → Extensions and enable it, then allow it on youtube.com.
  3. Click the puzzle icon in the toolbar, choose Open app, and set a scripts folder.
  4. Paste the script from the link below into a new file ending in .user.js.
Chrome, Edge, Firefox

Tampermonkey

  1. Install Tampermonkey (or Violentmonkey) from your browser's extension store.
  2. Open the raw script link below — the manager offers to install it automatically.
  3. Confirm, then reload youtube.com.
  4. Updates arrive on their own: the script carries an update URL.
